# WPUpYun 又拍云云存储插件

WordPress 又拍云云存储插件（简称：WPUPYUN），基于又拍云云存储与 WordPress 实现静态资源到又拍云对象存储中，实现网站数据与静态资源分离，提高网站加载速度。

## 功能特点

- **自定义域名**：支持自定义加速域名，可设置多级目录
- **一键禁止缩略图**：仅生成和上传主图，禁止缩略图裁剪
- **多站点支持**：支持自定义任意对象存储目录，一个存储桶可多网站使用
- **自动重命名**：上传文件自动重命名，解决中文文件名或重复文件名问题
- **本地与云端分离**：支持本地和对象存储分离、同步模式
- **不在本地保存**：可选仅上传至又拍云，不在服务器保留备份
- **删除同步**：删除媒体库附件时自动同步删除远程文件

## 环境要求

| 项目 | 要求 |
|------|------|
| WordPress | 5.3+ |
| PHP | 7.4+ |
| 又拍云 | 云存储服务、操作员账号 |

## 安装

1. 将 `wpupyun` 文件夹上传到 `/wp-content/plugins/` 目录
2. 在 WordPress 后台「插件」列表中激活 WPUpYun
3. 进入「设置」→「又拍云存储设置」配置参数

## 配置说明

在「又拍云存储设置」页面需填写：

| 配置项 | 说明 |
|--------|------|
| 服务名称 | 又拍云存储服务名称 |
| 绑定加速域名 | 自定义加速域名，如 `https://cdn.example.com` |
| 操作员用户 | 操作员用户名 |
| 授权操作员密码 | 操作员密码 |
| 自动重命名 | 开启后上传文件自动重命名 |
| 不在本地保存 | 开启后仅上传至又拍云，不保留本地副本 |
| 禁止缩略图 | 开启后仅上传主图，不生成缩略图 |

## 常见问题

**Q: 插件出错时如何排查？**  
A: 开启 WordPress 调试模式，查看 `wp-config.php` 中的 `WP_DEBUG` 与 `WP_DEBUG_LOG` 配置。

**Q: 是否支持本地与云端同时备份？**  
A: 支持。不勾选「不在本地保存」时，本地和又拍云都会保留文件。

**Q: 已有网站迁移到又拍云如何操作？**  
A: 先将本地静态资源上传到又拍云对应目录，再在插件中配置加速域名与账号，确保新旧 URL 使用同一域名和路径。

**Q: 建议使用前做什么？**  
A: 建议先完整备份网站和数据库，确认无误后再正式切换。
